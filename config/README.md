## ğŸ”§ Files Overview

### `env_config.dart`

* Auto-generated by [envied](https://pub.dev/packages/envied).
* Contains environment-specific values (e.g., API keys, endpoints).
* Do not edit manually â€” generated via `.env.{env}` files.

### `environment.dart`

* Defines the `Environment` enum: e.g., `dev`, `prod`.
* Helps differentiate logic for dev/prod builds.

```dart
enum Environment { dev, prod }
```

### `flavor.dart`

* Defines the `Flavor` enum: e.g., `free`, `paid`, `internal`.
* Enables behavior customization based on business variants.

```dart
enum Flavor { free, paid, internal }
```

### `project_config.dart`

* Singleton that exposes current `environment`, `flavor`, and optionally `envConfig` values.
* Use this as the main access point throughout the app.

### `project_flavor_loader.dart`

* Entry point to initialize both flavor and environment at app startup.
* Typically called in `main()` before running the app.

```dart
Future<void> main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await ProjectFlavorLoader.initialize(
    env: env,
    flavor: Flavor.internal,
  );
  runApp(MyApp());
}
```
